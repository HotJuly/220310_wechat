数据结构:
	[
		{
			id:"100",
			name:"国电榜",
			list:[
				{
					picUrl:"xxx.jpeg",
					name:"莲花谣",
					id:"1111"
				}
			]
		}
	]

分析过程:
	1.由于swiper中存储这多个排行榜,这些排行榜都是同类型数据
		所以选择使用数组进行存储多个榜单数据

	2.由于每个榜单中都具有自己的榜单名称和榜单列表,他们数据含义不相近,但是都是描述一个榜单的内容
		所以选择使用对象进行存储

	3.由于每个榜单都具有自己的歌曲排行榜列表,其中存储的都是歌曲信息,含义相近
		所以选择使用数组进行存储

	4.由于每首歌曲都具有自己的图片,排名,歌曲名称等信息,含义不相近,但是都是歌曲某些方向的描述
		所以选择使用对象进行存储



结构选型:
	1.对象
		key-value形式存储
		也就是说每个数据都具有他自己的唯一标识,或者说是独特的含义
		小总结:如果有一堆数据,这些数据都是用来描述某一个东西,那么就选择使用对象存储

	2.数组
		下标-value形式存储
		也就是说每个数据具有相似相近的含义
		小总结:如果有一堆数据,他们都是同类型数据,那么就选择使用数组存储
